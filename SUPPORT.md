# Support

Tri-State Relay Service is an early local developer tool. Support is best-effort and happens through GitHub issues when the repository is public.

## Good public issues

1. Reproducible bugs with app version, macOS version, install method, CLI path, expected behavior, actual behavior, and steps to reproduce.
2. Documentation corrections.
3. Feature requests that explain the problem, desired behavior, alternatives, and privacy or speaking-behavior impact.

## Do not post publicly

Do not paste secrets, access tokens, private notification text, relay queue databases, transcripts, customer data, crash reports with personal data, signing credentials, provisioning profiles, or full logs that may contain private paths or tokens.

For vulnerabilities or security-sensitive reports, follow `SECURITY.md`.

## Unsupported requests

The maintainer may close requests that require unattended speaking, remote control of playback, broad background automation, unsupported macOS versions, private infrastructure, or product directions outside the local direct-download app and CLI model.
