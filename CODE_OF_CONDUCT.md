# Code of Conduct

This project expects respectful, constructive participation.

## Standards

Examples of behavior that helps the project:

1. Assume good intent and stay focused on the product or documentation issue.
2. Keep bug reports and feature requests specific and reproducible.
3. Respect that this is a local tool that may involve private notifications and speech playback.
4. Avoid posting secrets, private data, or other people's content.

Unacceptable behavior includes harassment, personal attacks, sexualized language, sustained disruption, doxxing, publishing private information, or pressuring maintainers to accept unsafe changes.

## Enforcement

Maintainers may edit, hide, lock, or delete comments; close issues; block users; or restrict participation when needed to keep the project safe and useful.

Report conduct concerns through the same private path described in `SECURITY.md` until a dedicated contact is published.
